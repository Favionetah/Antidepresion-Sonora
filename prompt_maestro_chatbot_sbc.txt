OBJETIVO
--------
Desarrollar una aplicación de escritorio nativa para Windows (.exe), escrita en Python 3.10+, que implemente un Sistema Basado en Conocimiento (SBC) para el diagnóstico adaptativo de estrés y la prescripción personalizada de musicoterapia mediante Spotify.

El usuario interactuará mediante un chatbot conversacional que realizará preguntas, analizará síntomas mediante un árbol de decisión y recomendará playlists terapéuticas.

BASE DE CONOCIMIENTO
---------------------
Debe existir un único diccionario llamado:

    base_conocimiento

Debe contener exactamente 25 nodos.

Identificadores:
    N01
    N02
    ...
    N25

Cada nodo debe tener como mínimo:

    {
        "mensajes": [],
        "preguntas": [],
        "opciones": [],
        "spotify_playlist_id": "",
        "spotify_playlist_url": "",
        "playlist_nombre": "",
        "playlist_descripcion": "",
        "diagnostico": "",
        "recomendacion": "",
        "esHoja": False
    }

Toda la lógica debe surgir leyendo este diccionario.
Queda prohibido crear grandes bloques de if/elif/else.


MOTOR DE INFERENCIA
--------------------
El sistema debe funcionar mediante una Máquina de Estados Finitos.

Debe existir un estado actual.

Ejemplo:

    self.estado_actual = "N01"

Cuando el usuario pulse una opción:
- leer el nodo
- cambiar el estado
- actualizar el chat
- generar nuevos botones
- ejecutar Spotify si corresponde


PATRONES DE DISEÑO
--------------------

Singleton
---------
La Base de Conocimiento existirá una única vez.
Será inmutable.
Solo lectura.

State
-----
El comportamiento de la interfaz dependerá únicamente del nodo activo.

Nodo decisión:
- mostrar opciones

Nodo hoja:
- ocultar opciones
- mostrar diagnóstico
- ejecutar Spotify
- mostrar Reiniciar Diagnóstico

Command
-------
Cada botón generado dinámicamente encapsulará la acción de transición.

Ejemplo conceptual:

    lambda nodo=nodoDestino: self.cambiar_estado(nodo)
